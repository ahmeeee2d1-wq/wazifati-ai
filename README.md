# وظيفتي AI

منصة عربية شخصية لإدارة رحلة التوظيف: حساب حقيقي، تحليل وتحسين السيرة، مطابقة الوظائف، تجهيز التقديم، منع التكرار، متابعة الحالات، ومزامنة الحساب بين الأجهزة.

## التشغيل على هذا الجهاز

1. انقر مرتين على **افتح-الموقع.bat**.
2. سجّل الدخول بحسابك الحقيقي بعد ربط Supabase.
3. اترك نافذة التشغيل مفتوحة أثناء الاستخدام.

## النسخة السحابية الرسمية

المشروع جاهز للبناء والنشر عبر Vite وGitHub Pages، ويستخدم Supabase للحسابات وقاعدة البيانات والتخزين الخاص.

1. أنشئ مشروع Supabase وشغّل ملف `database/schema.sql` في SQL Editor.
2. أضف `VITE_SUPABASE_URL` و`VITE_SUPABASE_PUBLISHABLE_KEY` إلى أسرار مستودع GitHub.
3. فعّل GitHub Pages باستخدام GitHub Actions.
4. ملف `.github/workflows/deploy.yml` يبني وينشر الموقع تلقائيًا مع HTTPS.

عند وجود القيم السحابية يتحول تسجيل الدخول تلقائيًا من وضع التجربة إلى حسابات حقيقية، وتحفظ السيرة والملف والإعدادات والتقديمات في حساب المستخدم الخاص.

## ميزات الإنتاج

- واجهة عربية RTL متجاوبة مع الكمبيوتر والجوال.
- تطبيق PWA قابل للتثبيت على الشاشة الرئيسية.
- إنشاء حساب وتسجيل دخول وجلسة محفوظة.
- مزامنة البيانات بين الأجهزة.
- تخزين خاص للسيرة الذاتية حتى 10 ميجابايت.
- سياسات Row Level Security تمنع وصول المستخدمين إلى بيانات بعضهم.
- تحليل PDF النصي وDOCX وTXT محليًا ثم حفظ النتائج في الحساب.
- تحسين ATS وتصدير نسخة محسنة.
- مطابقة وظائف مع تفسير ودرجة ملاءمة.
- مراجعة قبل التقديم، حد يومي، ومنع تكرار على مستوى الواجهة وقاعدة البيانات.
- سجل حالات التقديمات.
- سياسة خصوصية وشروط استخدام.

## ملاحظة مهمة

البحث الحي والإرسال الخارجي ومتابعة البريد تحتاج موصلات رسمية لمصادر الوظائف والبريد. لا تدّعي المنصة إتمام إرسال خارجي من دون إثبات، ولا تنفذ تقديمًا عشوائيًا أو مخالفًا لشروط المصدر.

## الملفات المهمة

- `src/app.js`: الواجهة والتفاعلات.
- `src/cloud.js`: الحساب والمزامنة والتخزين السحابي.
- `src/cv-parser.js`: قراءة وتحليل السيرة.
- `src/matcher.js`: حساب ملاءمة الوظائف.
- `database/schema.sql`: قاعدة البيانات وسياسات الحماية.
- `.github/workflows/deploy.yml`: النشر التلقائي.
- `public/sw.js`: تثبيت التطبيق والعمل الموثوق على الجوال.
